# PRD.md — สเปคระบบติดตามการเพาะเลี้ยงเนื้อเยื่อพืช

## 1. ปัญหาและผู้ใช้
ติดตามจำนวนต้นพืชจากการเพาะเลี้ยงเนื้อเยื่อยาก ไม่รู้ว่าขวดไหนถึงรอบถ่ายโอนเมื่อไหร่ ต้องใช้สูตรอาหารอะไร
ผู้ใช้: เจ้าของสวน + ลูกจ้าง — **สิทธิ์การใช้งานเท่ากันทุกคน ไม่มีการแยกบทบาท**

## 2. โครงสร้างข้อมูล (Google Sheet)

### Sheet: `PlantTypes` (ชนิดพืช + การตั้งค่าระยะ)
| คอลัมน์ | ชนิดข้อมูล | คำอธิบาย |
|---|---|---|
| plant_id | text | รหัสอ้างอิง (auto) |
| plant_name | text | ชื่อชนิดพืช เช่น "กล้วยไม้ A" |
| stage1_rounds | number | จำนวนรอบของระยะขยาย ก่อนเข้าระยะกระตุ้นราก (วนซ้ำได้หลายรอบ) |
| stage1_days | number | จำนวนวันต่อรอบ ในระยะขยาย |
| stage1_recipe_tib | text | สูตรอาหารระยะขยาย — แบบ TIB (อาหารเหลว) |
| stage1_recipe_ss | text | สูตรอาหารระยะขยาย — แบบ SS (กึ่งแข็ง) |
| stage2_days | number | จำนวนวันระยะกระตุ้นราก (ทำรอบเดียวเสมอ) |
| stage2_recipe_tib | text | สูตรอาหารระยะกระตุ้นราก — แบบ TIB |
| stage2_recipe_ss | text | สูตรอาหารระยะกระตุ้นราก — แบบ SS |
| stage3_recipe_tib | text | สูตรอาหารระยะพร้อมออกปลูก — แบบ TIB (ระยะสุดท้าย) |
| stage3_recipe_ss | text | สูตรอาหารระยะพร้อมออกปลูก — แบบ SS |
| active | boolean | ใช้งานอยู่ / ปิดใช้งาน (ไม่ลบ) |

### Sheet: `Storage` (ที่เก็บ)
| คอลัมน์ | คำอธิบาย |
|---|---|
| shelf | ชั้น (1–4 หรือเพิ่มใหม่ได้) |
| sub_shelf | ช่อง A/B/C (เพิ่มใหม่ได้) |
| active | true/false |

### Sheet: `Transfers` (บันทึกการถ่ายโอนเนื้อเยื่อ — รายการหลัก)
| คอลัมน์ | คำอธิบาย |
|---|---|
| record_id | รหัสอ้างอิง (auto) |
| plant_id | อ้างถึง PlantTypes |
| plant_name | ชื่อชนิดพืช (เก็บซ้ำไว้กันชื่อเปลี่ยนทับของเก่า) |
| round_no | รอบที่ของขวดนี้ (1, 2, 3, ...) นับรวมทุกระยะ |
| transfer_date | วันที่บันทึกการถ่ายโอนครั้งนี้ |
| quantity | จำนวนต้นที่ทำได้ |
| media_type | TIB (อาหารเหลว) หรือ SS (กึ่งแข็ง) — เลือกใหม่ได้ทุกรอบ |
| bottle_no | หมายเลขขวด (จำเป็นเฉพาะตอน media_type = TIB) |
| shelf / sub_shelf | ตำแหน่งเก็บ |
| stage | ระยะปัจจุบันหลังถ่ายโอนครั้งนี้: ขยาย / กระตุ้นราก / พร้อมออกปลูก |
| recipe_used | สูตรอาหารที่ใช้รอบนี้ (ตามระยะ + media_type) |
| next_transfer_date | วันที่ถ่ายโอนครั้งถัดไป (คำนวณอัตโนมัติ, ถ้า stage = พร้อมออกปลูก ให้เว้นว่าง) |
| recorded_by | ชื่อผู้บันทึก (LINE display name) |
| created_at | timestamp |

### Sheet: `LineUsers` (ผู้รับแจ้งเตือน)
| คอลัมน์ | คำอธิบาย |
|---|---|
| line_user_id | ใช้สำหรับ push message (และใช้คัดลอกไปตั้งเป็น ADMIN_LINE_USER_ID ได้) |
| display_name | ชื่อที่แสดง |
| active | รับแจ้งเตือนอยู่หรือไม่ |

## 3. ตรรกะการคำนวณอัตโนมัติ (ฝั่ง Backend เท่านั้น)

เมื่อบันทึกการถ่ายโอน 1 ครั้ง:
1. ดู `round_no` ของขวดนี้ เทียบกับ `stage1_rounds` ที่ตั้งไว้ของชนิดพืชนั้น:
   - ถ้า `round_no <= stage1_rounds` → ยังอยู่ระยะ **ขยาย**
   - ถ้า `round_no == stage1_rounds + 1` → เข้าระยะ **กระตุ้นราก** (ทำรอบเดียวเสมอ)
   - ถ้า `round_no > stage1_rounds + 1` → เข้าระยะ **พร้อมออกปลูก** (ระยะสุดท้าย)
2. ดูว่ารอบนี้เลือก `media_type` เป็น TIB หรือ SS (เลือกใหม่ได้ทุกรอบ ไม่ผูกกับระยะ) แล้วดึง `recipe_used` ตามคู่ (ระยะ, media_type) ที่ตรงกัน
3. ถ้ายังไม่ถึงระยะสุดท้าย → คำนวณ `next_transfer_date` = วันที่บันทึก + จำนวนวันของระยะถัดไป (จากการตั้งค่า)
4. ถ้าเป็นระยะ "พร้อมออกปลูก" → `next_transfer_date` เว้นว่าง (จบรอบติดตาม)
5. ถ้า `media_type` = TIB → ต้องกรอก `bottle_no` (หมายเลขขวด) ด้วยเสมอ — ถ้าเป็น SS ไม่ต้องกรอก
6. ทุกครั้งที่บันทึกสำเร็จ → ส่งแจ้งเตือนทันทีไปยังแอดมิน (PI) ผ่าน Line (แยกจากแจ้งเตือนรายวัน) ถ้ามีการตั้งค่า `ADMIN_LINE_USER_ID` ไว้

## 4. สเปคแต่ละหน้า

### หน้า 0 — Line Rich Menu
- ปุ่ม "บันทึกถ่ายโอน" → เปิด `index.html` (LIFF)
- ปุ่ม "ตั้งค่า" → เปิด `settings.html` (LIFF)
- ปุ่ม "ดูสรุป" → เปิด `dashboard` (LIFF หรือลิงก์เว็บ)

### หน้า 1 — เลือกชนิดพืช (`liff-app/index.html`)
- แสดงรายการชนิดพืชที่ active เป็นปุ่มใหญ่ (มือถือ, กดง่าย)
- แต่ละปุ่มโชว์: ชื่อพืช + จำนวนขวดที่ถือครองอยู่ตอนนี้
- ปุ่มลอย "ดูสรุป" และ "ตั้งค่า" อยู่ล่างจอ
- กดเลือกชนิดพืช → ไปหน้า 2

### หน้า 2 — บันทึกการถ่ายโอน (`liff-app/record.html`)
ฟอร์มทางเดียว:
1. ชนิดพืช (ส่งมาจากหน้า 1 แล้ว, แก้ได้ผ่าน dropdown)
2. จำนวนต้นที่ทำได้ (ตัวเลข)
3. ประเภทรอบนี้: **TIB (อาหารเหลว)** หรือ **SS (กึ่งแข็ง)** — เลือกใหม่ได้ทุกรอบ
4. หมายเลขขวด — แสดงเฉพาะตอนเลือก TIB (จำเป็นต้องกรอก)
5. ชั้นเก็บ (เลือกจากปุ่ม 1–4 หรือชั้นที่เพิ่มเอง) + ช่อง A/B/C (เลือกจากปุ่ม หรือเพิ่มช่องใหม่)
6. ปุ่ม "บันทึก"

หลังกดบันทึก → แสดงผลทันที (ไม่เปลี่ยนหน้า):
> "บันทึกแล้ว ✓ ระยะ: กระตุ้นราก (รอบที่ 1/1) · สูตร: [ชื่อสูตร] · ประเภท: TIB (ขวดเลขที่ 014) · รอบถัดไป: 12 ก.ค. 69"

ปุ่ม "บันทึกขวดถัดไป" (เริ่มฟอร์มใหม่) และ "กลับหน้าหลัก"

ทุกครั้งที่บันทึกสำเร็จ ระบบส่งแจ้งเตือนทันทีไปยังแอดมิน (PI) ทาง Line โดยอัตโนมัติ (ไม่ต้องรอรอบแจ้งเตือนรายวัน)

### หน้า 3 — ตั้งค่า (`liff-app/settings.html`)
แท็บ 3 แท็บในหน้าเดียว:
1. **ชนิดพืช**: รายการชนิดพืช + ปุ่ม "เพิ่มชนิดพืชใหม่" → ฟอร์มแยกตามระยะ:
   - ระยะขยาย: จำนวนรอบ, จำนวนวันต่อรอบ, สูตร TIB, สูตร SS
   - ระยะกระตุ้นราก (รอบเดียวเสมอ): จำนวนวัน, สูตร TIB, สูตร SS
   - ระยะพร้อมออกปลูก (ระยะสุดท้าย): สูตร TIB, สูตร SS
   - แก้ไข/ปิดใช้งานได้ (ไม่มีปุ่มลบถาวร)
2. **ที่เก็บ**: รายการชั้น/ช่อง + ปุ่ม "เพิ่มชั้น" / "เพิ่มช่อง" + ปุ่มปิดใช้งานรายตัว
3. **ผู้รับแจ้งเตือน**: รายชื่อคนที่ผูก Line ไว้แล้ว พร้อม Line ID ที่กดคัดลอกได้ (ใช้สำหรับนำไปตั้งเป็นแอดมินในขั้น Apps Script)

### หน้า 4 — Dashboard (`dashboard/index.html`)
ไม่มีการกรอกข้อมูล ใช้ดูอย่างเดียว:
- **แถบสรุปบนสุด**: จำนวนขวดทั้งหมด, จำนวนต้นรวม, จำนวนขวดที่ถึงกำหนดถ่ายโอนใน 3 วัน
- **รายการ "ใกล้ถึงกำหนด"**: เรียงจากใกล้ที่สุด แสดงชนิดพืช + ระยะ/รอบที่ + ตำแหน่งเก็บ + ประเภท (TIB/SS) + หมายเลขขวด (ถ้ามี) + วันที่ + สูตรที่ต้องใช้รอบหน้า
- **ภาพรวมตามชนิดพืช**: การ์ดต่อชนิดพืช แสดงจำนวนขวดแยกตามระยะ (ขยาย/กระตุ้นราก/พร้อมออกปลูก) ด้วย stepper เขียวที่เห็นสัดส่วนได้ทันที
- **ภาพรวมที่เก็บ**: ตารางชั้น 1–4 × ช่อง A–C แสดงจำนวนขวดในแต่ละช่อง (ดูช่องที่แน่น/ว่างได้ไว)

### หน้า 5 — Line แจ้งเตือน (อัตโนมัติ, ไม่มีหน้าจอ)
มี 2 แบบ ทำงานแยกกัน:

**5a. แจ้งเตือนทันทีให้แอดมิน (PI)** — ทุกครั้งที่มีการบันทึกถ่ายโอนสำเร็จ
- ส่งให้คนเดียว ตาม `ADMIN_LINE_USER_ID` ที่ตั้งไว้ใน Script Properties เท่านั้น
- รูปแบบข้อความ:
```
📝 มีการบันทึกถ่ายโอนเนื้อเยื่อใหม่
กล้วยไม้ A — ขยาย (รอบที่ 2/3)
ประเภท: TIB · ขวดเลขที่ 014
ที่เก็บ: ชั้น 2/B · จำนวน 18 ต้น
บันทึกโดย: สมชาย
```

**5b. แจ้งเตือนรายวัน** — รันทุกเช้า (เช่น 7:00) เช็คขวดที่ `next_transfer_date` ตรงกับวันนี้หรือพรุ่งนี้
- ส่งข้อความสรุปทุกคนที่อยู่ใน `LineUsers` (active) — ทุกคนได้รับเหมือนกัน
- รูปแบบข้อความ:
```
🌱 แจ้งเตือนถ่ายโอนเนื้อเยื่อ
วันนี้/พรุ่งนี้ถึงกำหนด 2 ขวด:

• กล้วยไม้ A — ชั้น 2/B
  สูตรรอบหน้า: ระยะกระตุ้นราก (สูตร X)

• ว่านสี่ทิศ B — ชั้น 1/A
  สูตรรอบหน้า: ระยะพร้อมออกปลูก (สูตร Z)
```

### หน้า 6 — ประวัติการถ่ายโอน (`liff-app/history.html`)
ดูอย่างเดียว ไม่มีการกรอกข้อมูล ไล่เป็น 3 ขั้น:
1. เลือกชนิดพืช (เหมือนหน้า 1)
2. เลือกขวด — แสดงรายการขวดที่ "กำลังติดตามอยู่" ของชนิดนั้น (ตำแหน่งเก็บ + ระยะปัจจุบัน + รอบที่)
3. ไทม์ไลน์ — แสดงทุกรอบของขวดนั้นตั้งแต่รอบที่ 1 จนถึงปัจจุบัน เรียงตามลำดับ พร้อม: วันที่, ระยะ, ประเภท (TIB/SS) + หมายเลขขวด, จำนวนต้น, สูตรที่ใช้, ผู้บันทึก

เข้าถึงได้ 2 ทาง:
- จากหน้าหลัก ปุ่ม "📜 ดูประวัติการถ่ายโอน" (เริ่มจากขั้น 1)
- จากหน้าผลลัพธ์หลังบันทึกสำเร็จ ปุ่ม "📜 ดูประวัติขวดนี้" (ลิงก์ตรงไปขั้น 3 ของขวดที่เพิ่งบันทึก ผ่าน query string `?plant_id=...&shelf=...&sub_shelf=...`)

## 5. ขอบเขตที่ไม่รวมในเวอร์ชันนี้ (Out of scope)
- ไม่มีระบบ login/แยกสิทธิ์ผู้ใช้ (ทุกคนที่เข้าถึง Line OA ใช้งานได้เท่ากัน)
- ไม่มีการคำนวณต้นทุน/ค่าใช้จ่ายสูตรอาหาร
- ไม่มีแอปแยกนอก Line (ใช้ LIFF + Google Sheet เท่านั้น)
