# PRD.md — สเปคระบบติดตามการเพาะเลี้ยงเนื้อเยื่อพืช

## 1. ปัญหาและผู้ใช้
ติดตามจำนวนต้นพืชจากการเพาะเลี้ยงเนื้อเยื่อยาก ไม่รู้ว่าขวดไหนถึงรอบถ่ายโอนเมื่อไหร่ ต้องใช้สูตรอาหารอะไร
ผู้ใช้: ทีมงาน 3 ห้อง Lab (**KA**, **NJ**, **CC**) — แบ่งสิทธิ์ 2 ระดับ:
- **Admin**: เห็น/บันทึก/ดูประวัติ/Dashboard ได้ทุก Lab, ได้รับแจ้งเตือนทุกครั้งที่มีการบันทึกไม่ว่า Lab ไหน, ตั้งสิทธิ์ (role/Lab) ของผู้ใช้คนอื่นได้
- **User**: เห็น/บันทึก/ดูประวัติ/Dashboard/แจ้งเตือน **เฉพาะ Lab ของตัวเอง** — มองไม่เห็นข้อมูลของ Lab อื่นเลย

ชนิดพืช (PlantTypes), สูตรอาหาร, และชั้น/ช่องเก็บ (Storage) เป็นชุดข้อมูลกลางที่ใช้ร่วมกันทุก Lab (ไม่ได้แยกตาม Lab) — มีแค่ "การบันทึกถ่ายโอน" (Transfers) เท่านั้นที่แยกข้อมูลตาม Lab

## 3. ตรรกะการคำนวณอัตโนมัติ (ฝั่ง Backend เท่านั้น)

เมื่อบันทึกการถ่ายโอน 1 ครั้ง:
1. ดึง `lab` ของผู้บันทึกจากชีต `LineUsers` (อิง `line_user_id` ที่ส่งมาทุกครั้ง) — ถ้ายังไม่ได้รับอนุมัติจากแอดมิน (`approved` ยังเป็น false) ห้ามบันทึก, ถ้ายังไม่มี Lab (ยังไม่เคยเลือก) ห้ามบันทึก ต้องให้เปิดแอปใหม่เพื่อเลือก Lab ก่อน
2. ดู `round_no` ของล็อตนี้ (อ้างจาก `plant_id` + `lot_no` + `lab` — ไม่ใช่ตำแหน่งเก็บ และผูกกับ Lab ของผู้บันทึกเพื่อไม่ให้เลขล็อตชนกับ Lab อื่น) เทียบกับ `stage1_rounds` ที่ตั้งไว้ของชนิดพืชนั้น:
   - ถ้า `round_no <= stage1_rounds` → ยังอยู่ระยะ **ขยาย**
   - ถ้า `round_no == stage1_rounds + 1` → เข้าระยะ **กระตุ้นราก** (ทำรอบเดียวเสมอ)
   - ถ้า `round_no > stage1_rounds + 1` → เข้าระยะ **พร้อมออกปลูก** (ระยะสุดท้าย)
3. ดูว่ารอบนี้เลือก `media_type` เป็น TIB หรือ SS (เลือกใหม่ได้ทุกรอบ ไม่ผูกกับระยะ) แล้วดึง `recipe_used` ตามคู่ (ระยะ, media_type) ที่ตรงกัน
4. ถ้ายังไม่ถึงระยะสุดท้าย → คำนวณ `next_transfer_date` = วันที่บันทึก + จำนวนวันของระยะถัดไป (จากการตั้งค่า)
5. ถ้าเป็นระยะ "พร้อมออกปลูก" → `next_transfer_date` เว้นว่าง (จบรอบติดตาม)
6. ถ้า `media_type` = TIB → ต้องกรอก `bottle_no` (หมายเลขขวด) ด้วยเสมอ — ถ้าเป็น SS ไม่ต้องกรอก
7. ทุกครั้งที่บันทึกสำเร็จ → ส่งแจ้งเตือนทันทีผ่าน Line ให้ (ก) ผู้บันทึกเอง (ยืนยันว่าบันทึกสำเร็จ) และ (ข) ทุกคนที่มี role เป็น Admin (เห็นทุก Lab) — ไม่ส่งให้สมาชิกคนอื่นใน Lab เดียวกันที่ไม่ได้บันทึก
8. จำนวนต้น (`quantity`) ของรอบนี้ คำนวณจาก `bottle_count × plants_per_bottle` ที่ผู้ใช้กรอกมาทุกครั้งที่บันทึก (ไม่รับค่า quantity ที่ส่งมาตรงๆ จากฟอร์มอีกต่อไป)

## 4. สเปคแต่ละหน้า

### หน้า 0 — Line Rich Menu

### หน้า 0.4 — รออนุมัติจากแอดมิน (overlay เต็มจอ ใน `liff-app/js/api.js`)
- เกิดขึ้นอัตโนมัติตอนเปิดแอปครั้งแรกผ่าน Line (ทุกหน้า) ถ้าแอดมินยังไม่อนุมัติให้เข้าใช้งาน (`approved` = false) — บังหน้าจออื่นทั้งหมด ข้ามขั้นเลือก Lab ไปเลย (ยังเลือกไม่ได้จนกว่าจะอนุมัติ)
- ข้อความ: บอกว่าบัญชีเปิดใช้งานแล้วแต่รอแอดมินอนุมัติ + ปุ่ม "ตรวจสอบอีกครั้ง" (เรียก API ซ้ำเพื่อเช็กสถานะ ไม่ใช่ auto-poll พื้นหลัง)
- คนที่เคยใช้งานมาก่อนฟีเจอร์นี้ (มี Lab ตั้งไว้แล้ว) หรือ Admin จะถือว่าอนุมัติอัตโนมัติ ไม่เห็นหน้านี้
- แอดมินอนุมัติได้จากหน้า "สิทธิ์ผู้ใช้" (ดูหน้า 3) — กดปุ่ม "อนุมัติให้เข้าใช้งาน" หรือเลือก Lab ให้ตรงก็ถือเป็นการอนุมัติไปในตัว

### หน้า 0.5 — เลือก Lab ครั้งแรก (overlay เต็มจอ ใน `liff-app/js/api.js`)
- เกิดขึ้นอัตโนมัติตอนเปิดแอปครั้งแรกผ่าน Line (ทุกหน้า ไม่ใช่หน้าเฉพาะ) ถ้าได้รับอนุมัติแล้ว (ผ่านหน้า 0.4) แต่ยังไม่เคยเลือก Lab มาก่อน — บังหน้าจออื่นทั้งหมดจนกว่าจะเลือก
- แสดงปุ่มใหญ่ 3 ปุ่ม: **KA** / **NJ** / **CC**
- กดเลือกแล้วบันทึกทันที (ตั้งได้ครั้งเดียว) แล้วเข้าใช้งานหน้าที่ตั้งใจเปิดต่อตามปกติ
- ถ้าเลือกผิด/ต้องย้าย Lab ทีหลัง ต้องให้ Admin เปลี่ยนให้ผ่านหน้า "สิทธิ์ผู้ใช้" (ดูหน้า 3) ไม่มีปุ่มให้เปลี่ยน Lab ตัวเองอีกหลังจากเลือกแล้ว

### หน้า 1 — เลือกชนิดพืช (`liff-app/index.html`)
- แสดงรายการชนิดพืชที่ active เป็นปุ่มใหญ่ (มือถือ, กดง่าย)
- แต่ละปุ่มโชว์: ชื่อพืช + จำนวนล็อตที่ถือครองอยู่ตอนนี้ **ของ Lab ตัวเอง** (Admin เห็นรวมทุก Lab)
- badge มุมขวาบนแสดง Lab ปัจจุบัน (เช่น "Lab KA") หรือ "ทุก Lab" ถ้าเป็น Admin
- ปุ่มลอย "ดูสรุป" และ "ตั้งค่า" อยู่ล่างจอ
- กดเลือกชนิดพืช → ไปหน้า 2

ฟอร์มทางเดียว:
1. ชนิดพืช (ส่งมาจากหน้า 1 แล้ว, แก้ได้ผ่าน dropdown)
2. หมายเลขล็อต — เลือก **"ล็อตใหม่"** (พิมพ์หมายเลขล็อตใหม่) หรือ **"เลือกล็อตที่มีอยู่"** (เลือกจากรายการล็อตที่กำลังติดตามอยู่ของชนิดพืชนี้ใน Lab ตัวเอง พร้อมแสดงระยะ/รอบที่/ตำแหน่งล่าสุด/วันครบกำหนด เพื่อยืนยันก่อนเดินไปที่ชั้นเก็บจริง) — ฟิลด์นี้เป็นส่วนหนึ่งของฟอร์มเดียวกัน ไม่ถือเป็นขั้นที่ 5 ตามกฎ ≤4 ขั้น (เหมือนกับ media_type/bottle_no ที่เป็นรายละเอียดย่อยของ "ชนิดพืช/สิ่งที่บันทึก" ไม่ใช่ขั้นตอนนำทางใหม่)
3. จำนวนขวดที่ทำได้ (ตัวเลข, ค่าเริ่มต้น 20 ขวด ตามขนาดล็อต แก้ไขได้) + จำนวนต้นต่อขวด (ตัวเลข กรอกใหม่ทุกครั้ง ไม่มีค่าเริ่มต้น) — มีตัวเลขแสดงผลรวมเป็นจำนวนต้นให้ดูสดๆ ขณะกรอก แต่จำนวนต้นจริงที่บันทึกคำนวณที่ backend
4. ประเภทรอบนี้: **TIB (อาหารเหลว)** หรือ **SS (กึ่งแข็ง)** — เลือกใหม่ได้ทุกรอบ
5. หมายเลขขวด — แสดงเฉพาะตอนเลือก TIB (จำเป็นต้องกรอก)
6. ชั้นเก็บ (เลือกจากปุ่ม 1–4 หรือชั้นที่เพิ่มเอง) + ช่อง A/B/C (เลือกจากปุ่ม หรือเพิ่มช่องใหม่) — เป็นตำแหน่งของรอบนี้ อาจย้ายจากรอบก่อนได้
7. ปุ่ม "บันทึก"

หลังกดบันทึก → แสดงผลทันที (ไม่เปลี่ยนหน้า):
> "บันทึกแล้ว ✓ ล็อต 7 · ระยะ: กระตุ้นราก (รอบที่ 1/1) · สูตร: [ชื่อสูตร] · ประเภท: TIB (ขวดเลขที่ 014) · รอบถัดไป: 12 ก.ค. 69"

ปุ่ม "บันทึกอีกรายการ" (เริ่มฟอร์มใหม่) และ "กลับหน้าหลัก"

ทุกครั้งที่บันทึกสำเร็จ ระบบส่งแจ้งเตือนทันทีทาง Line ให้ผู้บันทึกเอง (ยืนยันว่าบันทึกสำเร็จ) และทุกคนที่เป็น Admin โดยอัตโนมัติ (ไม่ต้องรอรอบแจ้งเตือนรายวัน) — สมาชิกคนอื่นใน Lab เดียวกันที่ไม่ได้บันทึกจะไม่ได้รับ

### หน้า 3 — ตั้งค่า (`liff-app/settings.html`)
แท็บในหน้าเดียว — แท็บ "ชนิดพืช" และ "ที่เก็บ" ใช้ได้ทั้ง Admin และ User เหมือนกัน (เป็นข้อมูลกลางใช้ร่วมทุก Lab) ส่วนแท็บ "สิทธิ์ผู้ใช้" แสดงให้ Admin เท่านั้น (User เปิดมาจะไม่เห็นแท็บนี้เลย):
1. **ชนิดพืช**: รายการชนิดพืช + ปุ่ม "เพิ่มชนิดพืชใหม่" → ฟอร์มแยกตามระยะ:
   - ระยะขยาย: จำนวนรอบ, จำนวนวันต่อรอบ, สูตร TIB, สูตร SS
   - ระยะกระตุ้นราก (รอบเดียวเสมอ): จำนวนวัน, สูตร TIB, สูตร SS
   - ระยะพร้อมออกปลูก (ระยะสุดท้าย): สูตร TIB, สูตร SS
   - แก้ไข/ปิดใช้งานได้ (ไม่มีปุ่มลบถาวร)
2. **ที่เก็บ**: รายการชั้น/ช่อง + ปุ่ม "เพิ่มชั้น" / "เพิ่มช่อง" + ปุ่มปิดใช้งานรายตัว
3. **ระบบ SS/TIB** (เห็นเฉพาะ Admin): แก้ค่าพารามิเตอร์ระบบ SS/TIB ระดับระบบ (ชีต `SystemConfig`) — จำนวนวันต่อรอบ, ขวดสูงสุด/ชิ้นต่อขวด/ตัวคูณต่อรอบ ของระบบ SS และ TIB แยกกัน มีผลกับพยากรณ์อัตโนมัติที่ Dashboard (หน้า 4) ทันทีที่บันทึก และเป็นค่าเริ่มต้นของหน้า "จำลองรอบการผลิต" (หน้า 8 — แก้ในหน้าจำลองแต่ละครั้งได้โดยไม่กระทบค่ากลางนี้)
4. **สิทธิ์ผู้ใช้** (เห็นเฉพาะ Admin): รายชื่อคนที่เปิดแอปผ่าน Line ไว้แล้วทั้งหมด (ทุก Lab) แต่ละคนมี:
   - ป้าย "รออนุมัติ" ถ้ายังไม่ได้รับอนุมัติ + ปุ่ม "อนุมัติให้เข้าใช้งาน" (กดแล้วเลือก Lab เองได้ทันทีตอนเปิดแอปครั้งถัดไป)
   - ปุ่มเลือก Lab (KA/NJ/CC) — แตะเพื่อเปลี่ยน Lab ของคนนั้น (เปลี่ยน Lab ให้ถือว่าอนุมัติไปในตัวด้วย)
   - ปุ่มเลือกสิทธิ์ (User/Admin) — แตะเพื่อเปลี่ยนบทบาท
   - Line ID ที่กดคัดลอกได้ (เผื่อต้องใช้ตั้งเป็น `ADMIN_LINE_USER_ID` ตอนปลดล็อกแอดมินคนแรก)

### หน้า 4 — Dashboard (`dashboard/index.html`)
ไม่มีการกรอกข้อมูล ใช้ดูอย่างเดียว — **User เห็นแค่ข้อมูลของ Lab ตัวเอง, Admin เลือกได้ว่าจะดูรวมทุก Lab หรือดูเฉพาะ Lab เดียว** (badge มุมขวาบนแสดง "Lab KA/NJ/CC" หรือ "ทุก Lab"):
- **แถบเลือก Lab** (เห็นเฉพาะ Admin): ปุ่ม "ทุก Lab" / "KA" / "NJ" / "CC" — แตะเพื่อกรองข้อมูลทั้งหน้าให้เหลือแค่ Lab นั้น (ไม่ส่งค่า = ดูรวมทุก Lab เหมือนเดิม)
- **แถบสรุปบนสุด**: จำนวนล็อตทั้งหมด, จำนวนต้นรวม, จำนวนล็อตที่ถึงกำหนดถ่ายโอนใน 3 วัน
- **รายการ "ใกล้ถึงกำหนด"**: เรียงจากใกล้ที่สุด แสดงชนิดพืช + หมายเลขล็อต + ระยะ/รอบที่ + ตำแหน่งเก็บ + ประเภท (TIB/SS) + หมายเลขขวด (ถ้ามี) + วันที่ + สูตรที่ต้องใช้รอบหน้า + ชื่อผู้บันทึกรอบล่าสุด (ถ้าดูรวมทุก Lab จะมีคำว่า "Lab XX" นำหน้าแต่ละรายการ บอกว่าของ Lab ไหน)
- **สมาชิกแลป**: รายชื่อ + สิทธิ์ (User/Admin) ของคนที่อยู่ใน Lab ที่กำลังดู — ถ้า Admin เลือก "ทุก Lab" จะแยกเป็นกลุ่มตาม Lab ให้เห็นว่าใครอยู่แลปไหน
- **ภาพรวมตามชนิดพืช**: การ์ดต่อชนิดพืช แสดงจำนวนต้นแยกตามระยะ (ขยาย/กระตุ้นราก/พร้อมออกปลูก) ด้วย stepper เขียวที่เห็นสัดส่วนได้ทันที
- **พยากรณ์จำนวนต้น (ไม่บังคับ)**: ช่องกรอก "วันที่อยากได้เนื้อเยื่อ" เหนือการ์ด "ภาพรวมตามชนิดพืช" — ถ้ากรอก จะคำนวณจำนวนต้นที่พยากรณ์ได้ ณ วันนั้นต่อชนิดพืช (นับเฉพาะจำนวนต้นที่อยู่ระยะ "ขยาย" ของชนิดพืชนั้นเป็นชิ้นตั้งต้น เริ่มคำนวณจากวันนี้เสมอ) — ใช้ตัวคูณถ่วงน้ำหนักตามสัดส่วนความจุของระบบ SS:TIB (จากตั้งค่า "ระบบ SS/TIB") ทบต้นไปทุกรอบแบบไม่จำกัดความจุ (ไม่หัก "พร้อมออกปลูก" ออกระหว่างทางเหมือนหน้า 8 — เพราะที่นี่ต้องการดูแนวโน้มการเติบโตล้วนๆ ไม่ใช่จำลองกำลังผลิตจริงที่ติดเพดานความจุ SS/TIB) แสดงในการ์ดของชนิดพืชนั้นๆ ไม่แสดงเป็นตัวเลขรวมแยกต่างหาก
- **ภาพรวมที่เก็บ**: ตารางชั้น 1–4 × ช่อง A–C แสดงจำนวนล็อตในแต่ละช่อง (นับตามตำแหน่งล่าสุดที่บันทึกของแต่ละล็อต ภายในขอบเขต Lab ที่เห็นอยู่ — ดูช่องที่แน่น/ว่างได้ไว)

### หน้า 5 — Line แจ้งเตือน (อัตโนมัติ, ไม่มีหน้าจอ)
มี 2 แบบ ทำงานแยกกัน:

**5a. แจ้งเตือนทันที** — ทุกครั้งที่มีการบันทึกถ่ายโอนสำเร็จ
- ส่งให้ (1) ผู้บันทึกเอง (ยืนยันว่าบันทึกของตัวเองสำเร็จ) และ (2) ทุกคนที่ role เป็น Admin (เห็นทุก Lab) — ไม่ส่งให้สมาชิกคนอื่นใน Lab เดียวกันที่ไม่ได้บันทึก
- รูปแบบข้อความ:
```
📝 มีการบันทึกถ่ายโอนเนื้อเยื่อใหม่
Lab KA — กล้วยไม้ A — ล็อต 7 — ขยาย (รอบที่ 2/3)
ประเภท: TIB · ขวดเลขที่ 014
ที่เก็บ: ชั้น 2/B · จำนวน 18 ต้น
บันทึกโดย: สมชาย
```

**5b. แจ้งเตือนรายวัน** — รันทุกเช้า (เช่น 7:00) เช็คล็อตที่ `next_transfer_date` ตรงกับวันนี้หรือพรุ่งนี้ — **แยกข้อความตาม Lab**:
- สมาชิก User แต่ละ Lab ได้รับสรุปเฉพาะล็อตของ Lab ตัวเอง
- Admin ได้รับสรุปรวมทุก Lab เป็นข้อความเดียว (ไม่ได้รับซ้ำกับสรุปของ Lab ตัวเองอีก)
- รูปแบบข้อความ (ตัวอย่างของ User Lab KA):
```
🌱 แจ้งเตือนถ่ายโอนเนื้อเยื่อ — Lab KA
วันนี้/พรุ่งนี้ถึงกำหนด 2 ล็อต:

• กล้วยไม้ A — ล็อต 7 — ชั้น 2/B
  สูตรรอบหน้า: ระยะกระตุ้นราก (สูตร X)

• ว่านสี่ทิศ B — ล็อต 3 — ชั้น 1/A
  สูตรรอบหน้า: ระยะพร้อมออกปลูก (สูตร Z)
```
- รูปแบบข้อความของ Admin จะมี "Lab XX —" นำหน้าแต่ละบรรทัดแทน เพื่อบอกว่าของ Lab ไหน

### หน้า 6 — ประวัติการถ่ายโอน (`liff-app/history.html`)
ดูอย่างเดียว ไม่มีการกรอกข้อมูล (badge มุมขวาบนแสดง Lab ปัจจุบันเหมือนหน้าอื่น — เห็นเฉพาะล็อตของ Lab ตัวเอง ยกเว้น Admin เห็นได้ทุก Lab) ไล่เป็น 3 ขั้น:
1. เลือกชนิดพืช (เหมือนหน้า 1)
2. เลือกล็อต — แสดงรายการล็อตที่ "กำลังติดตามอยู่" ของชนิดนั้น (หมายเลขล็อต + ตำแหน่งเก็บล่าสุด + ระยะปัจจุบัน + รอบที่)
3. ไทม์ไลน์ — แสดงทุกรอบของล็อตนั้นตั้งแต่รอบที่ 1 จนถึงปัจจุบัน เรียงตามลำดับ พร้อม: วันที่, ระยะ, ตำแหน่งเก็บของรอบนั้นๆ, ประเภท (TIB/SS) + หมายเลขขวด, จำนวนต้น, สูตรที่ใช้, ผู้บันทึก

เข้าถึงได้ 2 ทาง:
- จากหน้าหลัก ปุ่ม "📜 ดูประวัติการถ่ายโอน" (เริ่มจากขั้น 1)
- จากหน้าผลลัพธ์หลังบันทึกสำเร็จ ปุ่ม "📜 ดูประวัติล็อตนี้" (ลิงก์ตรงไปขั้น 3 ของล็อตที่เพิ่งบันทึก ผ่าน query string `?plant_id=...&lot_no=...`) — ยังไม่ implement ในฟอร์มบันทึกตอนนี้ แต่ history.html รองรับ deep link นี้แล้ว

### หน้า 7 — วิธีใช้งาน (`liff-app/help.html`)
หน้าอ่านอย่างเดียว ไม่มีการเรียก API ที่แก้ข้อมูล (ใช้ `TissueApp.init()` แค่เพื่อแสดง badge Lab) สอนการใช้งานระบบแบบสั้นๆ เป็นการ์ดแยกหัวข้อ:
1. ภาพรวมระบบสั้นๆ (บันทึกถ่ายโอน → ดูสรุป/ประวัติ → รับแจ้งเตือนทาง Line)
2. ขั้นตอนบันทึกการถ่ายโอน (อ้างตามฟอร์มหน้า 2 — 4 ขั้นหลัก)
3. ความแตกต่าง Admin / User และเรื่อง Lab — รวมขั้นตอนตอนเปิดแอปครั้งแรก: ต้องรอแอดมินอนุมัติก่อน (หน้า 0.4) แล้วค่อยเลือก Lab (หน้า 0.5)
4. วิธีดูสรุปข้อมูล (Dashboard) และประวัติย้อนหลัง
5. เรื่องแจ้งเตือนทาง Line (ทันทีหลังบันทึก + รายวัน)

เข้าถึงได้จากปุ่ม "📖 วิธีใช้งาน" ในหน้าหลัก (`liff-app/index.html`)

### หน้า 8 — จำลองรอบการผลิต (`dashboard/simulate.html`)
เครื่องมือคำนวณ "ถ้าเริ่มจาก X ชิ้นวันนี้ จะได้กี่ต้นภายในวันที่ต้องการ" แยกอิสระจากข้อมูลล็อตจริงในระบบ (ไม่อ่าน/เขียน `Transfers`) ใช้วางแผนกำลังผลิตล่วงหน้าเท่านั้น เข้าถึงได้จากปุ่มในหน้า Dashboard (`dashboard/index.html`)

กรอกข้อมูลตั้งต้น:
- วันที่เริ่ม (ค่าเริ่มต้น = วันนี้), วันที่ต้องการเนื้อเยื่อ (ไม่บังคับ), จำนวนชิ้นตั้งต้น, จำนวนต้นเป้าหมาย (ไม่บังคับ — ต้องกรอกอย่างน้อย 1 ใน 2 อย่างระหว่างวันที่ต้องการ/จำนวนเป้าหมาย)
- จำนวนวันต่อรอบ, ระบบ SS (ขวดสูงสุด/ชิ้นต่อขวด/ตัวคูณต่อรอบ), ระบบ TIB (ขวดสูงสุด/ชิ้นต่อขวด/ตัวคูณต่อรอบ) — ค่าเริ่มต้นของทุกช่องนี้ดึงมาจากตั้งค่า "ระบบ SS/TIB" (หน้า 3) ตอนเปิดหน้านี้อัตโนมัติ แก้ในหน้านี้ได้เฉพาะครั้งที่กำลังจำลอง ไม่กระทบค่ากลาง
- อัตรารอดหลังออกปลูก (%) — สัดส่วนของส่วนเกินความจุที่ถูกนำออกไปกระตุ้นรากแล้วที่รอดจริงจนเป็นต้นพร้อมส่งมอบ (ส่วนที่ไม่รอดถือว่าสูญไปเลย ไม่กลับเข้าวงจรขยาย) — ค่าเริ่มต้น 100% ไม่ได้ดึงจากตั้งค่ากลาง — ไม่มีอัตรา/เกณฑ์ขั้นต่ำให้ตั้งเองอีกแล้ว เพราะตอนนี้ความจุของระบบ SS/TIB เป็นตัวกำหนดเองโดยอัตโนมัติว่าส่วนไหนถูกนำออกไปกระตุ้นราก (ดูตรรกะคำนวณด้านล่าง)

ตรรกะคำนวณ (ฝั่ง backend เท่านั้น ตามกฎเหล็ก — ดู `runProductionCycles` ใน `Code.gs`): ทุกรอบ แบ่งจำนวนชิ้นตั้งต้นเข้าระบบ SS และ TIB ตามสัดส่วนความจุของแต่ละระบบ (ความจุ = ขวดสูงสุด × ชิ้นต่อขวด) **ส่วนที่เกินความจุรวม (ระบบ SS/TIB เต็มแล้ว ไม่มีที่ลงต่อ) คือ "ส่วนเกิน" ที่ถูกนำออกไปกระตุ้นรากทันที** (ถ้าจำนวนชิ้นตั้งต้นยังไม่เกินความจุรวม ส่วนเกินจะเป็น 0 เสมอ — ไม่มีอะไรถูกนำออกไป ปล่อยขยายทบต้นต่อจนกว่าจะโตเกินความจุ) ส่วนที่โหลดเข้า SS/TIB ได้ คูณด้วยตัวคูณของระบบนั้น **เฉพาะผลคูณนี้เท่านั้น** ที่วนไปเป็นจำนวนชิ้นตั้งต้นของรอบถัดไป (ส่วนเกินไม่กลับเข้าวงจรขยายอีก)

การกระตุ้นรากใช้เวลา 1 รอบการผลิต — "ส่วนเกิน" ของรอบ N จะยังไม่กลายเป็น **"พร้อมออกปลูก"** ในรอบ N ทันที แต่ไปโผล่ใน **รอบ N+1** โดยคำนวณ = ส่วนเกินของรอบ N × อัตรารอดหลังออกปลูก (ส่วนที่ไม่รอดสูญไปเลย ไม่กลับเข้าวงจรขยาย) สะสมยอด "พร้อมออกปลูก" จากทุกรอบไว้ คำนวณต่อไปจนถึงวันที่ต้องการ หรือจนกว่ายอดสะสมนี้ถึงจำนวนเป้าหมาย แล้วหยุด — รอบสุดท้ายของการจำลองจะมี "ส่วนเกิน" ที่ยังไม่ทันแปลงเป็น "พร้อมออกปลูก" ค้างอยู่ (เพราะไม่มี "รอบถัดไป" ให้จำลองต่อ) แสดงแยกไว้ในสรุปท้ายตาราง ไม่รวมในยอดสะสม

แสดงผล:
- ตารางรายรอบ (เลื่อนแนวนอนได้บนมือถือ): รอบที่, วันที่, จำนวนชิ้นตั้งต้น, ความจุ/ขวดที่ใช้/ชิ้นที่โหลด/ผลคูณของ SS และ TIB, ส่วนเกิน (นำไปกระตุ้นราก), รวมปลายรอบ, พร้อมออกปลูก (รอบนี้ — มาจากส่วนเกินของรอบก่อนหน้า), พร้อมออกปลูกสะสม
- สรุปท้ายตาราง: จำนวนรอบทั้งหมด, พร้อมออกปลูกสะสมรวมทั้งหมด, จำนวนที่ยังค้างอยู่ระยะกระตุ้นราก (ถ้ามี — ของรอบสุดท้ายที่ยังไม่ทันแปลงเป็นพร้อมออกปลูก), ถึงจำนวนเป้าหมายหรือไม่ (เทียบกับยอดสะสม — ถ้าถึง บอกรอบที่/วันที่ที่ถึง, ถ้าไม่ถึง บอกว่าขาดอีกกี่ต้น)

## 5. ขอบเขตที่ไม่รวมในเวอร์ชันนี้ (Out of scope)
- ไม่มีระบบ login แบบใส่รหัสผ่าน — ระบุตัวตนด้วย LINE user ID จาก LIFF เท่านั้น (เชื่อ Line OA เป็นตัวยืนยันตัวตน)
- ไม่มีสิทธิ์ละเอียดกว่า Admin/User 2 ระดับ (เช่น "หัวหน้า Lab" ที่จัดการสิทธิ์ได้แค่ Lab ตัวเอง — ตอนนี้ Admin เห็น/จัดการได้ทุก Lab เท่านั้น ไม่มีสิทธิ์ระดับกลาง)
- ไม่มีการคำนวณต้นทุน/ค่าใช้จ่ายสูตรอาหาร
- ไม่มีแอปแยกนอก Line (ใช้ LIFF + Google Sheet เท่านั้น)
- ไม่มีการเติมหมายเลขล็อตหรือ Lab ให้กับข้อมูลเก่าก่อนฟีเจอร์นี้ (แถวเก่าที่ไม่มี `lot_no`/`lab` จะไม่แสดงในรายการล็อตที่ติดตามอยู่/Dashboard/แจ้งเตือนใดๆ แต่ยังอยู่ในชีตเดิม ไม่ถูกลบ — ดูวิธีจัดการตอน migrate ใน README.md)
- ไม่มีการย้ายล็อตข้าม Lab ผ่านหน้าจอ (ถ้าต้องย้ายข้อมูลข้าม Lab จริงๆ ต้องแก้ในชีตตรงๆ)
