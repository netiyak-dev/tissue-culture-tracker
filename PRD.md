# PRD.md — สเปคระบบติดตามการเพาะเลี้ยงเนื้อเยื่อพืช

## 1. ปัญหาและผู้ใช้
ติดตามจำนวนต้นพืชจากการเพาะเลี้ยงเนื้อเยื่อยาก ไม่รู้ว่าขวดไหนถึงรอบถ่ายโอนเมื่อไหร่ ต้องใช้สูตรอาหารอะไร
ผู้ใช้: เจ้าของสวน + ลูกจ้าง — **สิทธิ์การใช้งานเท่ากันทุกคน ไม่มีการแยกบทบาท**

## 2. โครงสร้างข้อมูล (Google Sheet)

### Sheet: `PlantTypes` (ชนิดพืช + การตั้งค่าระยะ)
| คอลัมน์ | ชนิดข้อมูล | คำอธิบาย |
|---|---|---|
| plant_id | text | รหัสอ้างอิง (auto) |
| plant_name | text | ชื่อชนิดพืช เช่น "กล้วยไม้ A" |
| stage1_days | number | จำนวนวันระยะขยาย |
| stage1_recipe | text | สูตรอาหารระยะขยาย |
| stage2_days | number | จำนวนวันระยะกระตุ้นราก |
| stage2_recipe | text | สูตรอาหารระยะกระตุ้นราก |
| stage3_recipe | text | สูตรอาหารระยะพร้อมออกปลูก (ระยะสุดท้าย ไม่มี days ถัดไป) |
| active | boolean | ใช้งานอยู่ / ปิดใช้งาน (ไม่ลบ) |

### Sheet: `Storage` (ที่เก็บ)
| คอลัมน์ | คำอธิบาย |
|---|---|
| shelf | ชั้น (1–4 หรือเพิ่มใหม่ได้) |
| sub_shelf | ช่อง A/B/C (เพิ่มใหม่ได้) |
| active | true/false |

### Sheet: `Transfers` (บันทึกการถ่ายโอนเนื้อเยื่อ — รายการหลัก)
| คอลัมน์ | คำอธิบาย |
|---|---|
| record_id | รหัสอ้างอิง (auto) |
| plant_id | อ้างถึง PlantTypes |
| plant_name | ชื่อชนิดพืช (เก็บซ้ำไว้กันชื่อเปลี่ยนทับของเก่า) |
| round_no | รอบที่ของขวดนี้ (1, 2, 3, ...) |
| transfer_date | วันที่บันทึกการถ่ายโอนครั้งนี้ |
| quantity | จำนวนต้นที่ทำได้ |
| shelf / sub_shelf | ตำแหน่งเก็บ |
| stage | ระยะปัจจุบันหลังถ่ายโอนครั้งนี้: ขยาย / กระตุ้นราก / พร้อมออกปลูก |
| recipe_used | สูตรอาหารที่ใช้รอบนี้ |
| next_transfer_date | วันที่ถ่ายโอนครั้งถัดไป (คำนวณอัตโนมัติ, ถ้า stage = พร้อมออกปลูก ให้เว้นว่าง) |
| recorded_by | ชื่อผู้บันทึก (LINE display name) |
| created_at | timestamp |

### Sheet: `LineUsers` (ผู้รับแจ้งเตือน)
| คอลัมน์ | คำอธิบาย |
|---|---|
| line_user_id | ใช้สำหรับ push message |
| display_name | ชื่อที่แสดง |
| active | รับแจ้งเตือนอยู่หรือไม่ |

## 3. ตรรกะการคำนวณอัตโนมัติ (ฝั่ง Backend เท่านั้น)

เมื่อบันทึกการถ่ายโอน 1 ครั้ง:
1. ดู `round_no` ของขวดนี้ (รอบที่ 1 = ขยาย, รอบที่ 2 = กระตุ้นราก, รอบที่ 3 ขึ้นไป = พร้อมออกปลูก) — จำนวนรอบต่อระยะอ่านจากการตั้งค่าของชนิดพืชนั้น
2. กำหนด `stage` และ `recipe_used` ตามระยะที่ตกอยู่
3. ถ้ายังไม่ถึงระยะสุดท้าย → คำนวณ `next_transfer_date` = วันที่บันทึก + จำนวนวันของระยะถัดไป (จากการตั้งค่า)
4. ถ้าเป็นระยะ "พร้อมออกปลูก" → `next_transfer_date` เว้นว่าง (จบรอบติดตาม)

## 4. สเปคแต่ละหน้า

### หน้า 0 — Line Rich Menu
- ปุ่ม "บันทึกถ่ายโอน" → เปิด `index.html` (LIFF)
- ปุ่ม "ตั้งค่า" → เปิด `settings.html` (LIFF)
- ปุ่ม "ดูสรุป" → เปิด `dashboard` (LIFF หรือลิงก์เว็บ)

### หน้า 1 — เลือกชนิดพืช (`liff-app/index.html`)
- แสดงรายการชนิดพืชที่ active เป็นปุ่มใหญ่ (มือถือ, กดง่าย)
- แต่ละปุ่มโชว์: ชื่อพืช + จำนวนขวดที่ถือครองอยู่ตอนนี้
- ปุ่มลอย "ดูสรุป" และ "ตั้งค่า" อยู่ล่างจอ
- กดเลือกชนิดพืช → ไปหน้า 2

### หน้า 2 — บันทึกการถ่ายโอน (`liff-app/record.html`)
ฟอร์มทางเดียว ไม่เกิน 4 ช่องกรอก:
1. ชนิดพืช (ส่งมาจากหน้า 1 แล้ว, แก้ได้ผ่าน dropdown)
2. จำนวนต้นที่ทำได้ (ตัวเลข)
3. ชั้นเก็บ (เลือกจากปุ่ม 1–4 หรือชั้นที่เพิ่มเอง) + ช่อง A/B/C (เลือกจากปุ่ม หรือเพิ่มช่องใหม่)
4. ปุ่ม "บันทึก"

หลังกดบันทึก → แสดงผลทันที (ไม่เปลี่ยนหน้า):
> "บันทึกแล้ว ✓ ระยะ: กระตุ้นราก · สูตร: [ชื่อสูตร] · รอบถัดไป: 12 ก.ค. 69"

ปุ่ม "บันทึกขวดถัดไป" (เริ่มฟอร์มใหม่) และ "กลับหน้าหลัก"

### หน้า 3 — ตั้งค่า (`liff-app/settings.html`)
แท็บ 3 แท็บในหน้าเดียว:
1. **ชนิดพืช**: รายการชนิดพืช + ปุ่ม "เพิ่มชนิดพืชใหม่" → ฟอร์ม (ชื่อ, จำนวนวันระยะขยาย, สูตรระยะขยาย, จำนวนวันระยะกระตุ้นราก, สูตรระยะกระตุ้นราก, สูตรระยะพร้อมออกปลูก) แก้ไข/ปิดใช้งานได้ (ไม่มีปุ่มลบถาวร)
2. **ที่เก็บ**: รายการชั้น/ช่อง + ปุ่ม "เพิ่มชั้น" / "เพิ่มช่อง"
3. **ผู้รับแจ้งเตือน**: รายชื่อคนที่ผูก Line ไว้แล้ว (เพิ่มเองผ่านการเปิด LIFF ครั้งแรกเท่านั้น ไม่มีฟอร์มกรอก ID มือ)

### หน้า 4 — Dashboard (`dashboard/index.html`)
ไม่มีการกรอกข้อมูล ใช้ดูอย่างเดียว:
- **แถบสรุปบนสุด**: จำนวนขวดทั้งหมด, จำนวนต้นรวม, จำนวนขวดที่ถึงกำหนดถ่ายโอนใน 3 วัน
- **รายการ "ใกล้ถึงกำหนด"**: เรียงจากใกล้ที่สุด แสดงชนิดพืช + ตำแหน่งเก็บ + วันที่ + สูตรที่ต้องใช้รอบหน้า
- **ภาพรวมตามชนิดพืช**: การ์ดต่อชนิดพืช แสดงจำนวนขวดแยกตามระยะ (ขยาย/กระตุ้นราก/พร้อมออกปลูก) ด้วย stepper เขียวที่เห็นสัดส่วนได้ทันที
- **ภาพรวมที่เก็บ**: ตารางชั้น 1–4 × ช่อง A–C แสดงจำนวนขวดในแต่ละช่อง (ดูช่องที่แน่น/ว่างได้ไว)

### หน้า 5 — Line แจ้งเตือน (อัตโนมัติ, ไม่มีหน้าจอ)
- รันทุกเช้า (เช่น 7:00) เช็คขวดที่ `next_transfer_date` ตรงกับวันนี้หรือพรุ่งนี้
- ส่งข้อความสรุปทุกคนที่อยู่ใน `LineUsers` (active)
- รูปแบบข้อความ:
```
🌱 แจ้งเตือนถ่ายโอนเนื้อเยื่อ
วันนี้/พรุ่งนี้ถึงกำหนด 2 ขวด:

• กล้วยไม้ A — ชั้น 2/B
  สูตร: ระยะกระตุ้นราก (สูตร X)

• ว่านสี่ทิศ B — ชั้น 1/A
  สูตร: ระยะพร้อมออกปลูก (สูตร Z)
```

## 5. ขอบเขตที่ไม่รวมในเวอร์ชันนี้ (Out of scope)
- ไม่มีระบบ login/แยกสิทธิ์ผู้ใช้ (ทุกคนที่เข้าถึง Line OA ใช้งานได้เท่ากัน)
- ไม่มีการคำนวณต้นทุน/ค่าใช้จ่ายสูตรอาหาร
- ไม่มีแอปแยกนอก Line (ใช้ LIFF + Google Sheet เท่านั้น)
