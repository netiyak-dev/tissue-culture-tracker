# CLAUDE.md — กฎเหล็กของโปรเจกต์

> ไฟล์นี้คือกฎที่ Claude (หรือนักพัฒนาคนใดก็ตาม) ต้องทำตามทุกครั้งที่แก้หรือเพิ่มโค้ดในโปรเจกต์นี้
> อ่าน `brand.md` (น้ำเสียง) และ `PRD.md` (สเปคแต่ละหน้า) คู่กับไฟล์นี้เสมอ

## ภาพรวมระบบ
ระบบติดตามการเพาะเลี้ยงเนื้อเยื่อพืช สำหรับสวนขนาดเล็ก (เจ้าของ + ลูกจ้าง สิทธิ์เท่ากัน)
- **บันทึกข้อมูล**: ผ่าน Line Rich Menu → เปิดฟอร์ม LIFF (เว็บที่เด้งในแอป Line)
- **ฐานข้อมูล**: Google Sheet (ผ่าน Google Apps Script เป็น backend/API)
- **ดูสรุปข้อมูล**: Dashboard (เว็บ) + เปิด Google Sheet ตรงได้
- **แจ้งเตือน**: Line Messaging API ส่งล่วงหน้าเมื่อใกล้ถึงวันถ่ายโอน

## กฎเหล็ก — สี (ห้ามใช้สีอื่นนอกจากชุดนี้)

| ชื่อ | Hex | ใช้ตรงไหน |
|---|---|---|
| Forest (Primary) | `#1B4332` | หัวข้อ, ปุ่มหลัก, แถบบน, ตัวเลขสำคัญ |
| Leaf (Secondary) | `#40916C` | ปุ่มรอง, สถานะ active, ไอคอน, ลิงก์ |
| Mint (Surface) | `#D8F3DC` | พื้นหลังการ์ด, แถบ stage ที่ผ่านแล้ว |
| Background | `#FAFAF7` | พื้นหลังหน้าจอหลัก |
| Text Primary | `#1B1E1C` | ข้อความหลัก |
| Text Muted | `#6B7280` | ข้อความรอง, label, คำอธิบาย |
| Border | `#D9DDD9` | เส้นขอบ, ตัวแบ่ง |
| Alert (ใช้น้อยที่สุด) | `#B65C1F` | แจ้งเตือนถึงกำหนด/เกินกำหนดเท่านั้น |

ห้าม: ห้ามเพิ่มสีไล่เฉด (gradient) แฟนซี ห้ามใช้สีนีออน/สีจัด ห้ามใช้สีแดงเข้มแบบ error ทั่วไป (ใช้ Alert ที่กำหนดแทน)

## กฎเหล็ก — ฟอนต์

- ใช้ **Sarabun** (Google Fonts) ตัวเดียวทั้งระบบ ทุกหน้า ทุก weight ที่ใช้: 400 (ข้อความ), 600 (หัวข้อ/ปุ่ม), 700 (ตัวเลขเด่น/ชื่อหน้า)
- ห้ามผสมฟอนต์อื่นเข้ามา รวมถึงฟอนต์ default ของระบบปฏิบัติการ
- ขนาดตัวอักษรขั้นต่ำบนฟอร์มมือถือ: 16px (กันเบราว์เซอร์ซูมอัตโนมัติ)

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Sarabun:wght@400;600;700&display=swap" rel="stylesheet">
```

## กฎเหล็ก — ภาษา

- ทุกตัวอักษรที่ผู้ใช้เห็นต้องเป็น**ภาษาไทย 100%** — ฟอร์ม, ปุ่ม, ข้อความแจ้งเตือน, Dashboard, ข้อความผิดพลาด
- ห้ามมีคำอังกฤษปนในสิ่งที่ผู้ใช้เห็น ยกเว้น: ชื่อเฉพาะที่ไม่มีคำแปล (เช่น ชื่อพันธุ์พืชภาษาอังกฤษที่ผู้ใช้กรอกเอง)
- ใช้คำศัพท์ตามตาราง "Vocabulary" ใน `brand.md` เท่านั้น ห้ามสลับคำ (เช่น ห้ามใช้ "transfer" ปนกับ "ถ่ายโอน")
- รูปแบบวันที่: `D MMM YY` แบบไทย (เช่น "12 ก.ค. 69") ใช้ปี พ.ศ.

## ข้อห้ามเชิงโครงสร้าง (Hard Constraints)

1. **ห้าม** ให้ฟอร์มบันทึกการถ่ายโอน 1 ครั้งมีมากกว่า 4 ขั้นตอน (เลือกพืช → จำนวนต้น → ที่เก็บ → บันทึก)
2. **ห้าม** ออกแบบแบบ desktop-first — ทุกหน้าต้องออกแบบให้ใช้งานในจอ LIFF มือถือ (~380–420px) เป็นอันดับแรก แล้วค่อยขยายจอใหญ่
3. **ห้าม** ลบแถวข้อมูลใน Google Sheet โดยไม่มีการยืนยันจากผู้ใช้ก่อนเสมอ (แก้ไข/ปิดใช้งานได้ แต่ห้ามลบเงียบๆ)
4. **ห้าม** ฮาร์ดโค้ดชนิดพืช จำนวนวันต่อระยะ หรือสูตรอาหารไว้ในโค้ด — ทุกอย่างต้องตั้งค่าได้จากหน้า "ตั้งค่า" โดยไม่ต้องพึ่งช่างไอที
5. **ห้าม** ใช้ third-party UI framework ขนาดใหญ่ (เช่น React/Bootstrap เต็มชุด) — ใช้ HTML/CSS/JS ธรรมดา เพื่อให้โหลดเร็วในเว็บวิวของ Line
6. **ห้าม** เก็บ token/secret (LINE Channel Access Token, Channel Secret) ไว้ในไฟล์ฝั่ง frontend — ต้องอยู่ใน Google Apps Script Properties เท่านั้น
7. ทุกการคำนวณอัตโนมัติ (ระยะการเจริญเติบโต / สูตรอาหาร / วันถ่ายโอนครั้งถัดไป) ต้องทำที่ฝั่ง backend (Apps Script) ไม่ใช่ฝั่งหน้าจอ เพื่อกันข้อมูลเพี้ยนถ้าเปิดพร้อมกันหลายคน

## โครงสร้างโปรเจกต์

```
tissue-culture-tracker/
├── brand.md              # น้ำเสียงแบรนด์
├── CLAUDE.md              # ไฟล์นี้ — กฎเหล็ก
├── PRD.md                 # สเปคแต่ละหน้า
├── liff-app/               # ฟอร์ม LIFF (เปิดผ่าน Line)
│   ├── index.html          # หน้าหลัก: เลือกชนิดพืช
│   ├── record.html         # ฟอร์มบันทึกการถ่ายโอน
│   ├── settings.html       # ตั้งค่าชนิดพืช/สูตรอาหาร/ที่เก็บ
│   ├── css/style.css
│   └── js/api.js
├── dashboard/               # Dashboard สรุปข้อมูล
│   ├── index.html
│   ├── css/style.css
│   └── js/dashboard.js
└── backend/
    └── Code.gs             # Google Apps Script: API + คำนวณ + แจ้งเตือน Line
```

## เมื่อแก้ไขโค้ดในอนาคต

- เพิ่มหน้าใหม่ → ต้องอัปเดต `PRD.md` ก่อน แล้วค่อยเขียนโค้ด
- เปลี่ยนคำในหน้าจอ → ต้องตรงกับ `brand.md` ก่อนเสมอ
- เปลี่ยนสี/ฟอนต์ → ต้องแก้ตารางในไฟล์นี้ก่อน ไม่แก้ใน CSS เพี้ยนๆ แบบจุดเดียว
