# brand.md — น้ำเสียงแบรนด์ (Brand Voice)

ระบบ: **ระบบติดตามการเพาะเลี้ยงเนื้อเยื่อพืช**
ผู้ใช้: เจ้าของสวน + ลูกจ้าง (สิทธิ์เท่ากันทุกคน)

## ตัวตนของระบบ
ระบบนี้คือ **"ผู้ช่วยประจำโรงเรือน"** — ไม่ใช่แอปไอทีโชว์เท่ แต่เป็นเครื่องมือทำงานที่ใช้ทุกวัน
เปรียบเหมือนสมุดบันทึกของสวนที่ฉลาดขึ้น จำแทนคนได้ คำนวณแทนคนได้ และเตือนแทนคนได้
แต่ยังคงความเรียบง่ายแบบสมุดจดงานสวน ไม่ใช่ระบบซับซ้อนแบบซอฟต์แวร์องค์กรใหญ่

## หลักการพูด (Tone Principles)

1. **ตรงประเด็น ไม่อ้อม** — บอกสิ่งที่ต้องทำ ไม่ใช่อธิบายระบบทำงานอย่างไร
   - ใช้: "บันทึกแล้ว ✓ รอบถัดไป 12 ก.ค. 69"
   - ไม่ใช้: "ระบบได้ประมวลผลและจัดเก็บข้อมูลของท่านเรียบร้อยแล้ว"

2. **ภาษาคนสวน ไม่ใช่ภาษาไอที** — ห้ามใช้ศัพท์เทคนิคที่ลูกจ้างอ่านไม่เข้าใจ
   - ใช้: "ขวดนี้", "ชั้นเก็บ", "รอบถ่ายโอน", "สูตรอาหารที่ต้องใช้"
   - ไม่ใช้: "record", "batch ID", "sync ข้อมูล", "trigger อัตโนมัติ"

3. **สุภาพ กระชับ เหมือนคนในทีมบอกกัน** — ไม่ใช่ทางการแบบหนังสือราชการ ไม่ใช่กันเองจนดูไม่น่าเชื่อถือ
   - ใช้: "วันนี้มีขวดที่ต้องถ่ายโอน 3 ขวด"
   - ไม่ใช้: "เรียน ผู้ใช้งาน แจ้งให้ทราบว่า..." (เป็นทางการเกิน)
   - ไม่ใช้: "เฮ้! อย่าลืมนะ 😆" (กันเองเกิน)

4. **บอกสิ่งที่ต้องทำ ก่อนบอกสถานะ** — ข้อความแจ้งเตือนต้อง action-first
   - ใช้: "พรุ่งนี้ถ่ายโอน: กล้วยไม้ A ชั้น 2/B — สูตรกระตุ้นราก"
   - ไม่ใช้: "สถานะ: กล้วยไม้ A อยู่ในระยะกระตุ้นราก กำหนดถ่ายโอนพรุ่งนี้"

5. **ไม่อวด ไม่ขายของ** — ไม่ใช้คำโฆษณา ไม่ใส่ feature ที่ไม่ได้ขอ ไม่ป้อนคำชมระบบตัวเอง

## โทนข้อความแต่ละจุด

| จุดที่ใช้ | โทน | ตัวอย่าง |
|---|---|---|
| ฟอร์มบันทึกข้อมูล | กระชับ ใช้ปุ่ม ไม่ใช้พารากราฟยาว | "เลือกชนิดพืช" / "บันทึก" |
| ข้อความสำเร็จ | ยืนยันสั้นๆ + บอกสิ่งที่เกิดต่อ | "บันทึกแล้ว ✓ รอบถัดไป 12 ก.ค. 69" |
| Line แจ้งเตือน | บอกสิ่งที่ต้องทำวันนี้/พรุ่งนี้ตรงๆ | "วันนี้ถึงกำหนดถ่ายโอน 2 ขวด: ..." |
| ข้อความผิดพลาด | บอกว่าผิดอะไร แก้ยังไง ไม่ใช้คำว่า "error" | "กรุณาใส่จำนวนต้นก่อนบันทึก" |
| หน้า Dashboard | สรุปเป็นตัวเลข/รายการ ไม่ใช่บทความ | "รออยู่ 4 ขวด · ถ่ายโอนแล้ว 32 ขวด" |
| หน้าตั้งค่า | อธิบายผลลัพธ์ของการตั้งค่า ไม่ใช่กลไกเบื้องหลัง | "ระยะนี้นาน 30 วัน ก่อนขึ้นระยะถัดไป" |

## คำที่ใช้เป็นมาตรฐาน (Vocabulary)

ใช้คำเดิมตลอดทั้งระบบ ห้ามสลับคำไปมา:
- "ถ่ายโอนเนื้อเยื่อ" (ไม่ใช้ "transfer", "subculture")
- "ขวด" (หน่วยของหนึ่งรายการที่ติดตาม)
- "ชั้นเก็บ" + "ช่อง A/B/C" (ไม่ใช่ "ตำแหน่ง", "location", "slot")
- "ระยะขยาย" / "ระยะกระตุ้นราก" / "พร้อมออกปลูก" (3 ระยะเท่านั้น ห้ามเพิ่มคำเรียกอื่น)
- "สูตรอาหาร" (ไม่ใช่ "formula", "media", "recipe")
- "รอบถ่ายโอนครั้งถัดไป" (ไม่ใช่ "next cycle", "due date")
- "TIB" (อาหารเหลว) / "SS" (กึ่งแข็ง) — ใช้ตัวย่อภาษาอังกฤษตรงนี้เท่านั้น เพราะเป็นชื่อเฉพาะทางวิชาการที่ผู้ใช้คุ้นเคยอยู่แล้ว แต่ต้องมีคำขยายภาษาไทยติดไว้เสมอ (เช่น "TIB (อาหารเหลว)")
- "หมายเลขขวด" (ไม่ใช่ "bottle ID", "serial number")
- "แอดมิน (PI)" (ผู้ดูแลระบบที่ได้รับแจ้งเตือนทันทีทุกครั้งที่มีการบันทึก)
